# Data Model: Rune-Langium Native Code Generators

**Feature**: `015-rune-codegen-zod`
**Phase**: 1 (Design)
**Date**: 2026-04-26

This document defines the entities introduced by this feature —
both the spec-level domain entities (from `spec.md §Key Entities`)
and the internal implementation types required by the generator
pipeline. Existing monorepo entities (workspace records, telemetry
events, curated manifests) are unchanged.

---

## §1 — Generator (domain entity)

A pure function that accepts one or more parsed Langium documents
and returns a `GeneratorOutput` tree. The Generator has no I/O
of its own; all I/O is delegated to the caller (the CLI writes to
disk; the Studio integration writes to in-memory strings).

**Shape**:

```ts
// packages/codegen/src/index.ts
export function generate(
  documents: LangiumDocument | LangiumDocument[],
  options: GeneratorOptions
): GeneratorOutput[];
```

**Invariants**:
- Calling `generate` with identical `documents` content and identical
  `options` MUST return byte-identical output (SC-007). This requires
  deterministic iteration: types are emitted in topological-sort
  order (see `TypeReferenceGraph`), not in `Map` insertion order.
- `generate` MUST NOT perform any network or disk I/O.
- `generate` MUST NOT mutate the input `documents` or their AST nodes.
- If any document has unresolved cross-references, `generate` emits
  a `GeneratorDiagnostic` with `severity: 'error'` but still
  attempts partial generation for well-formed namespaces.

**Relationships**:
- Receives `LangiumDocument[]` from `@rune-langium/core` (Langium
  services; the generator does not own the parse step).
- Produces `GeneratorOutput[]` — one per Rune namespace in the input.
- Reads `GeneratorOptions.target` to select the emitter pipeline.

## §2 — GeneratorOutput (domain entity)

A single emitted output unit. One `GeneratorOutput` is produced per
Rune namespace (one namespace = one output file for most targets).

**Shape**:

```ts
// packages/codegen/src/types.ts
export interface GeneratorOutput {
  /** Relative path for the output file (e.g., "cdm/base/math.zod.ts"). */
  relativePath: string;
  /** The full text of the emitted file. */
  content: string;
  /**
   * Source-map entries: each entry maps a zero-based output line
   * to the originating Rune source location.
   *
   * Populated for all three targets (FR-018).
   */
  sourceMap: SourceMapEntry[];
  /**
   * Generator-time diagnostics for this namespace.
   * Non-fatal diagnostics (e.g., reserved-word rename) are included
   * alongside output; fatal diagnostics (e.g., unresolved attribute
   * in a condition) set severity: 'error'.
   */
  diagnostics: GeneratorDiagnostic[];
}
```

**Invariants**:
- `relativePath` follows the Rune package hierarchy (e.g.,
  `cdm.base.math` → `cdm/base/math.zod.ts` for the Zod target,
  `cdm/base/math.schema.json` for JSON Schema, `cdm/base/math.ts`
  for TypeScript).
- `sourceMap` is never `null`; it may be empty for generated preamble
  lines (e.g., the `// generated by rune-codegen` header).
- `diagnostics` may be non-empty even when `content` is non-empty
  (partial generation is allowed for non-fatal diagnostics).

**Relationships**:
- Produced by `Generator`.
- Consumed by the CLI (writes `content` to `{outputDir}/{relativePath}`)
  and by the Studio integration (renders `content` in the preview
  panel; uses `sourceMap` for click-to-navigate).

## §3 — Target (domain entity)

A discriminated identifier selecting which emitter pipeline runs.

**Shape**:

```ts
// packages/codegen/src/types.ts
export type Target = 'zod' | 'json-schema' | 'typescript';
```

**Semantics**:
- `'zod'` (canonical / default): emits `*.zod.ts` files containing
  `z.object()` schemas with `.superRefine()` condition closures and
  `z.infer<>` type aliases. Runtime dep: `zod@^4.3.x` for consumers.
- `'json-schema'`: emits `*.schema.json` files conforming to JSON
  Schema 2020-12. No runtime dep for consumers.
- `'typescript'`: emits `*.ts` files with class declarations, type
  guards, `from()` static factories, and `validate*()` condition
  instance methods. No runtime Zod dep for consumers.

**Relationships**: Stored in `GeneratorOptions.target`. The Studio
target-switcher UI maps to the three values of this type.

## §4 — Fixture (domain entity)

A `.rune` source file paired with expected output and JSON test cases,
used for Tier 1 fixture-diff tests and Tier 2 JSON battery.

**Shape**:

```ts
// Conceptual — not a runtime type; describes the on-disk structure
interface Fixture {
  /** The .rune source input. */
  source: string; // path: test/fixtures/{name}/input.rune
  /** Expected generator output (committed). */
  expectedZod: string;       // test/fixtures/{name}/expected.zod.ts
  expectedJsonSchema: string; // test/fixtures/{name}/expected.schema.json
  expectedTs: string;        // test/fixtures/{name}/expected.ts
  /**
   * JSON test cases for behavioral correctness.
   * Each case is labeled valid or invalid and may carry a message
   * substring the error output must contain.
   */
  jsonCases: Array<{
    label: string;
    payload: unknown;
    valid: boolean;
    messageContains?: string; // for invalid cases
  }>; // path: test/fixtures/{name}/cases.json
}
```

**Fixture taxonomy** (FR-022):

| Directory | Constructs covered |
|-----------|-------------------|
| `basic-types/` | scalar attributes (string, int, date, boolean), empty type |
| `cardinality/` | all five cardinality forms: (1..1), (0..1), (0..*), (1..*), (n..m) |
| `enums/` | enum without displayName, enum with displayName, reserved-word enum member |
| `inheritance/` | single-level extend, multi-level chain, discriminator predicates |
| `conditions-simple/` | one-of, choice, exists, is-absent, only-exists |
| `conditions-complex/` | if/then/else, path navigation, arithmetic/comparison, aggregations, higher-order (filter/map), disjoint |
| `circular/` | mutual cycle (A→B→A), self-reference (Foo→Foo), three-way cycle |
| `reserved-words/` | attribute named `class`, `default`, `enum`, `function` |
| `meta-types/` | basicType, annotation, corpus declarations |
| `key-refs/` | as-key / key-reference cross-type pointers |

**Invariants**:
- For every fixture, `generate(parse(source), { target: 'zod' })`
  output MUST equal `expectedZod` byte-for-byte (SC-007, FR-022).
- The `jsonCases` array MUST contain at least one valid case and at
  least one invalid case for any fixture that has conditions.

## §5 — TypeReferenceGraph (internal type)

The directed graph used by the cycle detector and topological sorter.
Not exposed in the public API.

**Shape**:

```ts
// packages/codegen/src/cycle-detector.ts (internal)
interface TypeReferenceGraph {
  /** All type names in the namespace. */
  nodes: string[];
  /**
   * Directed edges: if type A has an attribute of type B,
   * there is an edge A → B.
   */
  edges: Map<string, string[]>;
}
```

**Construction rules**:
1. Walk every `Data` node in the document set.
2. For each `Attribute`, if its type resolves to another `Data` node
   (not a Langium built-in), add an edge from the containing type to
   the attribute's type name.
3. `extends` relationships also contribute edges: `Child extends
   Parent` adds edge `Child → Parent`.
4. `Enumeration` nodes are NOT included in the graph (they cannot
   participate in cycles).

**Usage**:
- Tarjan's SCC algorithm runs over `TypeReferenceGraph` to find
  cyclic type sets (see R3 in `research.md`).
- Kahn's algorithm runs over the condensation (SCC-collapsed DAG) to
  produce the emit order.

**Invariants**:
- Every `Data` type name in the input documents MUST appear in `nodes`.
- `edges` is a `Map` (not a plain object) to preserve deterministic
  iteration order (keys inserted in document-source order).

## §6 — EmissionContext (internal type)

Accumulated state threaded through the emitter for a single namespace.
Not exposed in the public API.

**Shape**:

```ts
// packages/codegen/src/emit/zod-emitter.ts (and siblings; internal)
interface EmissionContext {
  /** The target being emitted. */
  target: Target;
  /** Sorted emit order for this namespace. */
  emitOrder: string[];
  /** Set of type names that require z.lazy() wrapping. */
  lazyTypes: Set<string>;
  /** Map of original attribute names to their safe TS identifiers. */
  renamedKeys: Map<string, string>; // e.g., 'class' → 'class_'
  /**
   * Collected source-map entries; populated as each type/attribute
   * is emitted via toStringAndTrace().
   */
  sourceMap: SourceMapEntry[];
  /**
   * Generator-time diagnostics accumulated during this emission.
   */
  diagnostics: GeneratorDiagnostic[];
}
```

**Invariants**:
- `emitOrder` is derived from the topological sort result; it must
  cover exactly the types in the namespace (no extras, no omissions).
- `lazyTypes` is derived from Tarjan's SCC pass; a type in `lazyTypes`
  MUST emit a `z.lazy(() => …)` wrapper (or the TS-target equivalent
  late-binding pattern).
- `renamedKeys` is populated before emission begins; the same key
  always maps to the same renamed identifier (SC-007 determinism).

## §7 — ExpressionTranspilerContext (internal type)

State passed to the expression transpiler for a single `condition`
block. Not exposed in the public API.

**Shape**:

```ts
// packages/codegen/src/expr/transpiler.ts (internal)
interface ExpressionTranspilerContext {
  /**
   * The name of the `this` value in the emitted predicate.
   * In Zod mode: `val` (the `.superRefine(val, ctx) =>` parameter).
   * In TS-method mode: `this` (the class instance).
   */
  selfName: string;
  /**
   * How to emit errors.
   * 'zod-refine': push to `ctx.issues` via `ctx.addIssue({...})`.
   * 'ts-method':  push to local `errors: string[]`.
   */
  emitMode: 'zod-refine' | 'ts-method';
  /**
   * The name of the condition being transpiled (for error messages).
   */
  conditionName: string;
  /**
   * The type name the condition is attached to (for error messages).
   */
  typeName: string;
  /**
   * Map of attribute names to their types (for existence checks).
   * Derived from the parent Data node's attribute list.
   */
  attributeTypes: Map<string, string>;
}
```

**Usage**: `transpileExpression(runeExpr, ctx)` returns a JavaScript
expression string. For condition blocks, `transpileCondition(cond, ctx)`
wraps the expression in a guard and emits error accumulation.

**Invariants**:
- `attributeTypes` MUST be populated from the actual `Data` node before
  the transpiler is invoked (FR-025: mis-spelled references detected
  here — if the condition references an attribute not in `attributeTypes`,
  a `GeneratorDiagnostic` with `severity: 'error'` is emitted and the
  predicate is replaced with `/* DIAGNOSTIC: unknown attribute "foo" */`
  which fails `tsc --noEmit` deterministically).

## §8 — RuntimeHelperBundle (internal type)

The source strings for the three inlined runtime helpers, packaged
as a constant so all emitters share the same canonical text.
Not exposed in the public API.

**Shape**:

```ts
// packages/codegen/src/helpers.ts (internal)
export const RUNTIME_HELPER_SOURCE = `
// --- rune-codegen runtime helpers (inlined) ---
const runeCheckOneOf = (values: (unknown | undefined | null)[]): boolean =>
  values.filter(v => v !== undefined && v !== null).length === 1;

const runeCount = (arr: unknown[] | undefined | null): number =>
  arr?.length ?? 0;

const runeAttrExists = (v: unknown): boolean =>
  v !== undefined && v !== null && !(Array.isArray(v) && v.length === 0);
// --- end runtime helpers ---
`.trim();
```

**Invariants**:
- `runeCheckOneOf`: accepts an arbitrary-length array of optional
  values; returns `true` iff exactly one is non-null/undefined.
  Mirrors the Python generator's `rune_check_one_of` semantics
  (SC-003 parity contract).
- `runeCount`: returns the length of an array attribute, treating
  `undefined`/`null` as 0 (mirrors `rune_count`).
- `runeAttrExists`: returns `true` if the value is neither absent
  (`undefined`/`null`) nor an empty array. Mirrors `rune_attr_exists`.
  The empty-array case covers Rune's `(0..*)` attributes that are
  initialized to `[]` rather than `null`.
- These strings are injected at the top of every emitted file
  (after the SPDX header and imports). They are `const` arrow
  functions so TypeScript's dead-code elimination can tree-shake
  them in consumer bundles if unused.

## §9 — SourceMapEntry (supporting type)

**Shape**:

```ts
// packages/codegen/src/types.ts
export interface SourceMapEntry {
  /** Zero-based line in the emitted output file. */
  outputLine: number;
  /** URI of the originating .rune source document. */
  sourceUri: string;
  /** One-based line in the .rune source (Langium convention). */
  sourceLine: number;
  /** One-based character offset in the .rune source. */
```

## §10 — GeneratorDiagnostic (supporting type)

**Shape**:

```ts
// packages/codegen/src/types.ts
export interface GeneratorDiagnostic {
  severity: 'error' | 'warning' | 'info';
  message: string;
  /** Source file where the problem originates (if traceable). */
  sourceUri?: string;
  /** One-based line in the .rune source. */
  line?: number;
  /** One-based character offset. */
  char?: number;
  /**
   * Machine-readable code for tooling.
   * Examples: 'unknown-attribute', 'reserved-word-rename', 'cycle-detected'
   */
  code: string;
}
```

## §11 — GeneratorOptions (supporting type)

**Shape**:

```ts
// packages/codegen/src/types.ts
export interface GeneratorOptions {
  /** Which emitter pipeline to use. Defaults to 'zod'. */
  target?: Target;
  /**
   * If true, treat any generator-time 'error' diagnostic as a
   * thrown exception rather than a non-empty diagnostics array.
   * Useful for CLI --strict mode (FR-025).
   */
  strict?: boolean;
  /**
   * When set, prefix every emitted file's header comment with this
   * string (e.g., a timestamp or version for reproducibility
   * testing). Not included in byte-identical determinism check
   * (SC-007) — callers that want determinism MUST NOT set this.
   */
  headerComment?: string;
}
```

## Entity Relationship Summary

```
                     ┌──────────────────┐
                     │     Generator    │
                     │  generate(docs,  │
                     │    options)      │
                     └────────┬─────────┘
                              │ produces (1..*)
                              ▼
                   ┌──────────────────────┐
                   │   GeneratorOutput    │
                   │  relativePath        │
                   │  content             │
                   │  sourceMap[]         │
                   │  diagnostics[]       │
                   └──────────────────────┘
                              ▲
                              │ built by
              ┌───────────────┼───────────────┐
              │               │               │
    ┌──────────────┐  ┌───────────────┐  ┌──────────────┐
    │ ZodEmitter   │  │JsonSchemaEmit │  │  TsEmitter   │
    └──────────────┘  └───────────────┘  └──────────────┘
         all use ──────────────────────────────►
                   ┌─────────────────────────┐
                   │  ExpressionTranspiler   │
                   │  (shared, mode-switched)│
                   └─────────────────────────┘
                         reads from
                   ┌─────────────────────────┐
                   │  TypeReferenceGraph     │
                   │  (Tarjan SCC + Kahn)    │
                   └─────────────────────────┘
                         produces
          ┌──────────────┬──────────────────────┐
          │              │                      │
  ┌──────────────┐  ┌──────────────┐  ┌──────────────────┐
  │ EmissionCtx  │  │  ExprTxCtx   │  │RuntimeHelperBundle│
  └──────────────┘  └──────────────┘  └──────────────────┘
```

**Fixture** is a test-only entity (no runtime representation); it
relates to the Generator in the same direction as the spec domain:
`Fixture → Generator → GeneratorOutput → JSON assertion`.
